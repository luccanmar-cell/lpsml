Ingles:

Requirements:

Python 3.10 or newer
A terminal such as PowerShell

What each file is for

build_dataset.py
Converts the raw Excel file into a processed parquet dataset.

data_processing_utils.py
Contains the data-loading and preprocessing logic used by build_dataset.py.

model_training.py
Trains the models, evaluates them, saves predictions, and produces scored datasets.

training_utils.py
Handles configuration loading, train/test splitting, and model search.

metrics_utils.py
Computes metrics, saves plots, and writes JSON/Excel outputs.

launch_dashboard.py
Starts the Streamlit error dashboard.

error_dashboard.py
Visualizes prediction errors from the scored dataset.

Create and activate a virtual environment

On Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1```

If PowerShell blocks the activation script, run this once in the same terminal:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
```
Install dependencies

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Place your raw Excel file in the project root. The default expected file is:

ADS1436160159821391600.xlsx

Convert it into a processed parquet dataset:

```powershell
python build_dataset.py ADS1436160159821391600.xlsx -o ADS1436160159821391600.parquet
```

This script will:
read the Excel file,
build a modeling-ready dataset,
save a clean parquet file,
save a separate doubtful rows parquet file (for review).

The main file to use for training is the clean parquet output, usually: ADS1436160159821391600.parquet


Run the training pipeline with the processed parquet file:

```powershell
python model_training.py ADS1436160159821391600.parquet --search grid
```

```powershell
python model_training.py .\ADS1436160159821391600.parquet --search optuna
```

This step will create a new folder inside the training_runs directory with a timestamp, for example:

training_runs/20260717_111503_523873/

Inside that folder you will find:
run_summary.json
scored_dataset.parquet
scored_dataset.xlsx
a subfolder per model type containing model.joblib and metrics.json

Open the dashboard with the scored parquet generated by the training run.

```powershell
python launch_dashboard.py .\training_runs\20260717_111503_523873\scored_dataset.parquet
```

The dashboard expects a scored parquet file produced by model_training.py, it will open a Streamlit app.

Espanol:

Requisitos:

Python 3.10 o superior
Una terminal como PowerShell

Función de cada archivo

build_dataset.py
Convierte el archivo Excel original en un conjunto de datos procesado en formato parquet.

data_processing_utils.py
Contiene la lógica de carga y preprocesamiento de datos utilizada por build_dataset.py.

model_training.py
Entrena los modelos, los evalúa, guarda las predicciones y genera conjuntos de datos con puntuaciones.

training_utils.py
Gestiona la carga de configuración, la división de datos en entrenamiento/prueba y la búsqueda de modelos.

metrics_utils.py
Calcula métricas, guarda gráficos y genera archivos de salida en formato JSON o Excel.

launch_dashboard.py
Inicia el panel de control (dashboard) de errores en Streamlit.

error_dashboard.py
Visualiza los errores de predicción del conjunto de datos puntuado.

Crear y activar un entorno virtual

En Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1```

Si PowerShell bloquea el script de activación, ejecuta esto una vez en la misma terminal:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
```
Instalar dependencias

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Coloca tu archivo Excel original en la raíz del proyecto.

ADS1436160159821391600.xlsx

Conviértelo en un conjunto de datos procesado en formato parquet:

```powershell
python build_dataset.py ADS1436160159821391600.xlsx -o ADS1436160159821391600.parquet
```

Este script realizará las siguientes acciones:
leerá el archivo Excel,
creará un conjunto de datos listo para el modelado,
guardará un archivo parquet limpio,
guardará un archivo parquet separado con las filas dudosas (para su revisión). El archivo principal que se debe utilizar para el entrenamiento es el archivo Parquet limpio resultante, generalmente: `ADS1436160159821391600.parquet`

Ejecuta la canalización de entrenamiento con el archivo Parquet procesado:

```powershell
python model_training.py ADS1436160159821391600.parquet --search grid
```

```powershell
python model_training.py .\ADS1436160159821391600.parquet --search optuna
```

Este paso creará una nueva carpeta dentro del directorio `training_runs` con una marca de tiempo; por ejemplo:

training_runs/20260717_111503_523873/

Dentro de esa carpeta encontrarás:
`run_summary.json`
`scored_dataset.parquet`
`scored_dataset.xlsx`
una subcarpeta por tipo de modelo que contiene `model.joblib` y `metrics.json`

Abre el panel de control (dashboard) con el archivo Parquet con puntuaciones generado durante la ejecución:

```powershell
python launch_dashboard.py .\training_runs\20260717_111503_523873\scored_dataset.parquet
```

El panel de control espera un archivo Parquet con puntuaciones generado por `model_training.py`, se abrirá una aplicación Streamlit.
